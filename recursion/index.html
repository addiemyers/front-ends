What is an explanation page?

An explanation page is a document whose purpose is to make something unfamiliar feel familiar. It takes a thing — an idea, a phenomenon, a process — and breaks it into smaller, digestible pieces, then reassembles them in the reader's mind.

At its core, every explanation page has the same three jobs: introduce a subject, build understanding through structure, and leave the reader changed — knowing something they didn't before.

Explaining is the act of translating a mental model from one mind to another. But to do that, we need to talk about what a mental model is

A mental model is your internal map of how something works
. To talk about that, we need to understand how the brain builds concepts. To understand that, we need a bit of cognitive science. To understand that…

You see the pattern forming. To fully explain anything, you eventually need to explain everything. This is the fundamental problem of explanation: the chain has no natural beginning.

pseudocode
function explain(concept) { const prerequisites = getPrerequisites(concept) for (const prereq of prerequisites) { explain(prereq) // ← recursive call } return describe(concept) } // explain("sky is blue") // → explain("light") // → explain("photons") // → explain("quantum mechanics") // → explain("mathematics") // → explain("logic") // → explain("axioms") // → ∞

Good explanation pages sidestep this by choosing an assumed baseline — a place where they stop digging and say "you already know this." Where to draw that line is the author's central design decision.

This is recursion.

What we just did — explaining explanation by explaining the explaining of explanation — is exactly what recursion is: a process that calls itself as part of its own definition.

In computing, a recursive function is one that calls itself. In thinking, a recursive concept is one that refers to itself in its own definition. This page is a recursive object: an explanation page that explains itself.

call #4 — explain(baseCase)

Every recursion needs
a base case.

Without a stopping condition, recursion spirals forever. The base case is the clause that says: "if we're here, we don't go deeper — we already know." It's what makes recursive processes finite and useful rather than infinite and catastrophic.

function explain(concept) { // base case — stop here if (audienceAlreadyUnderstands(concept)) { return // ← no deeper call } // recursive case const simpler = simplify(concept) explain(simpler) // ← go one level deeper return buildOn(simpler, concept) }

For this explanation page, the base case is you. We assume you can read. We assume you know what a browser is. We stopped there. Those are our axioms — the things we don't explain, because we trust you already have them.
